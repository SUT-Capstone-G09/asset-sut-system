# สรุปงานแก้ไขระบบ — Asset SUT System

**Branch:** `space-availability-check`
**ช่วงเวลา:** เมื่อวาน–วันนี้ (2 วัน)
**รวม:** 22 จุดที่แก้ไข ใน 10 หมวดหมู่ (backend Go + frontend Next.js)

ทุกจุดผ่านการทดสอบ build (`go build` / `go vet` / `go test`, `tsc --noEmit`) และยิงทดสอบ API/หน้าเว็บจริงก่อนสรุปว่าแก้เสร็จ ไม่ใช่แค่แก้ตามที่อ่านโค้ดเดา

## สารบัญ

1. ระบบ build ใช้งานไม่ได้เลย (4 จุด)
2. สิทธิ์การใช้งานของพนักงาน (staff) (1 จุด)
3. ข้อมูลอาคารไม่เคยถูกบันทึกจริง (1 จุด)
4. ค่า "ศูนย์" ที่ถูกต้อง แต่ระบบดันปฏิเสธ (2 จุด)
5. การตั้งค่าระบบ (config) (2 จุด)
6. ฟอร์มสร้างเอกสารขออนุญาตใช้สถานที่ (3 จุด)
7. เอกสารแนบตอนยืนยันการจอง (1 จุด)
8. ลายเซ็นดิจิทัล — รวมช่องโหว่ความปลอดภัย (2 จุด)
9. ปรับปรุงหน้าตาและการใช้งาน (UX) (5 จุด)
10. ตัดการพึ่งพา CDN ภายนอก (1 จุด)
11. [วิธีไปทำต่อที่เครื่องอื่น](#การไปทำต่อที่เครื่องอื่น)

## 1. ระบบ build ใช้งานไม่ได้เลย

### `npm run build` พังทันที

- ปัญหา: bundler ตัวใหม่ (Turbopack) หา library บางตัวไม่เจอในโปรเจกต์นี้ ทั้งที่โหมด dev ใช้ปกติ เพราะ dev สั่งใช้ bundler คนละตัวกัน
- แก้แล้ว: ปักหมุดให้ build ใช้ bundler เดียวกับ dev — ผ่านครบ 49 หน้า

### มีไฟล์เศษหลงเหลือทำให้ build พัง

- ปัญหา: ไฟล์ component เก่าที่ไม่มีใครเรียกใช้แล้ว ถูกวางผิดตำแหน่งจนกลายเป็นหน้าเว็บโดยไม่ตั้งใจ
- แก้แล้ว: ลบทิ้ง

### Type ผิดสะสมทั่วระบบ ~9 จุด

- ปัญหา: เช่น ส่ง id ผิดชนิดข้อมูล, ส่ง field ที่ไม่มีอยู่จริงใน API, หน้าค้นหาห้องไม่รองรับการโหลดแบบ static
- แก้แล้ว: ไล่แก้ทีละจุดจนไม่มี error เหลือ

### หน้าแรกพังทุกครั้งที่โหลด

- ปัญหา: รูปภาพตัวอย่างในหน้าแรกใช้ URL ที่มีอักษรไทยแปะดิบๆ โดยไม่เข้ารหัส ทำให้ browser สร้าง header สำหรับ preload รูปไม่ได้
- แก้แล้ว: เข้ารหัส URL ให้ถูกต้อง

## 2. สิทธิ์การใช้งานของพนักงาน (staff)

### พนักงาน login เข้าได้ แต่กดอะไรก็ไม่ได้เลย

- ปัญหา: ตอน seed ข้อมูลเริ่มต้น ระบบให้สิทธิ์ (permission) กับบัญชี admin เท่านั้น บัญชี staff ไม่เคยได้รับสิทธิ์อะไรเลยสักอย่าง
- แก้แล้ว: เพิ่มสิทธิ์ให้ staff ครบทุกงาน ยกเว้นการจัดการผู้ใช้ ซึ่งสงวนไว้ให้ admin เท่านั้น

## 3. ข้อมูลอาคารไม่เคยถูกบันทึกจริง

### ระบุอาคารตอนสร้าง/แก้ไขห้องและโถง แล้วหายไปเงียบๆ

- ปัญหา: ซ้อนกันอยู่ 3 ชั้น — ฐานข้อมูลไม่เคยมีรายชื่ออาคารจริงเลย, ฟอร์มห้องส่งข้อมูลผิด field ไปให้ API, ฟอร์มโถงใช้รายชื่ออาคารปลอมที่ไม่ตรงกับอาคารจริงในระบบแม้แต่ตัวเดียว
- แก้แล้ว: แก้ครบทั้ง 3 ชั้น ตอนนี้เลือกอาคารแล้วบันทึกได้จริง ดึงจากฐานข้อมูลจริง

## 4. ค่า "ศูนย์" ที่ถูกต้อง แต่ระบบดันปฏิเสธ

### สร้างโถง (hall) ไม่ได้เลยสักครั้งเดียว

- ปัญหา: โถงไม่มีแนวคิดเรื่อง "ความจุ" จึงส่งค่า 0 ไปเสมอ แต่ระบบตั้งกฎไว้ผิดว่าค่า 0 = "ไม่ได้กรอก" เลยปัดตกทุกครั้ง ทั้งที่ 0 คือค่าที่ถูกต้องอยู่แล้ว
- แก้แล้ว: แก้กฎการตรวจสอบให้ยอมรับ 0

### ตั้งราคาห้องแบบให้ใช้ฟรี (0 บาท) ไม่ได้

- ปัญหา: ติดกฎเดียวกับข้อบน
- แก้แล้ว: แก้จุดเดียวกันทั้งสองที่

## 5. การตั้งค่าระบบ (config)

### Log ของระบบยาวเป็นหน้ากระดาษทุกครั้งที่รัน

- ปัญหา: ตัวแปรตั้งค่าระดับความละเอียดของ log สะกดไม่ตรงกับที่โค้ดอ่าน ทำให้การตั้งค่า "แสดงเฉพาะ error" ไม่เคยมีผลจริง ระบบพิมพ์ log ทุกคำสั่งฐานข้อมูลตลอดเวลา
- แก้แล้ว: แก้ชื่อตัวแปรให้ตรงกัน

### สร้าง QR รับชำระเงินไม่ได้เลย

- ปัญหา: ไฟล์ตั้งค่าเครื่องทดสอบขาดเลขบัญชีพร้อมเพย์ที่จำเป็นสำหรับสร้าง QR
- แก้แล้ว: เติมค่าที่ขาดแล้วสร้าง QR ได้ปกติ

## 6. ฟอร์มสร้างเอกสารขออนุญาตใช้สถานที่

### ช่องกรอก "วัตถุประสงค์" พิมพ์อะไรก็หายทันที

- ปัญหา: ลืมเชื่อมช่องกรอกกับตัวจัดการค่าที่พิมพ์ ทำให้ทุกตัวอักษรที่พิมพ์ถูกล้างทิ้งทันที
- แก้แล้ว: เชื่อมสายให้ถูกต้อง พิมพ์ได้ปกติ

### ช่องเลือกอาคารมีตัวเลือกที่ไม่ตรงกับความจริงเลยสักอาคาร

- ปัญหา: ตัวเลือกอาคารที่ตั้งไว้ล่วงหน้าไม่ตรงกับอาคารจริงในระบบเลยสักตัว ผู้ใช้ต้องกด "อื่นๆ" แล้วพิมพ์เองทุกครั้งอยู่ดี
- แก้แล้ว: ตัดตัวเลือกที่ไม่มีประโยชน์ออก เหลือช่องกรอกตรงพร้อมชื่อห้องที่จองไว้ให้อัตโนมัติ

### เพิ่มช่องกรอกเลขประจำตัวผู้เสียภาษี (ฟีเจอร์ใหม่)

- ใส่ในฟอร์มและในเอกสาร PDF ที่สร้างออกมาด้วย สำหรับหน่วยงาน/บริษัทที่ต้องใช้ออกใบกำกับภาษี

## 7. เอกสารแนบตอนยืนยันการจอง

### เลือก "อัปโหลดเอกสาร" หรือ "สร้างเอกสารผ่านระบบ" อันเดียว แต่ติ๊กถูกทั้งคู่

- ปัญหา: ระบบเก็บสถานะ "ใช้วิธีไหน" แยกจากลิสต์ไฟล์จริง ทำให้พอสร้างเอกสารสำเร็จ ทั้งสองการ์ดจะติ๊กถูกพร้อมกันเสมอ
- แก้แล้ว: ปรับให้แต่ละไฟล์จำที่มาของตัวเองว่าอัปโหลดเองหรือสร้างจากระบบ พร้อมเพิ่ม label บอกที่มาให้เห็นในลิสต์ไฟล์ด้วย

## 8. ลายเซ็นดิจิทัล

### บางครั้งลายเซ็นที่เคยบันทึกไว้ "หายไป" ดื้อๆ

- ปัญหา: ระบบมองทุกความผิดพลาด (เน็ตสะดุด, เซิร์ฟเวอร์รีสตาร์ท ฯลฯ) เป็นแบบเดียวกับ "ยังไม่เคยบันทึกลายเซ็น" หมด โดยไม่แจ้งอะไรเลย
- แก้แล้ว: แยกสองกรณีออกจากกัน ถ้าโหลดพลาดจริงจะมีข้อความแจ้งเตือนพร้อมปุ่ม "ลองใหม่" ทั้งในหน้าโปรไฟล์และฟอร์มสร้างเอกสาร

### 🔒 [ความปลอดภัย] อัปโหลดไฟล์ปลอมอ้างว่าเป็น PNG หลอกระบบได้

- ปัญหา: ฝั่งเซิร์ฟเวอร์เชื่อแค่ป้ายกำกับที่ผู้ใช้ส่งมาเอง ไม่เคยเปิดไฟล์ตรวจสอบจริง — ทดสอบแล้วอัปโหลดไฟล์ข้อความธรรมดาแปะป้าย PNG ผ่านได้จริง
- แก้แล้ว: เพิ่มการตรวจไฟล์จริงระดับไบต์ (ลายเซ็นไฟล์ PNG) และตรวจพื้นหลังโปร่งใสจริงฝั่งเซิร์ฟเวอร์ ทดสอบซ้ำยืนยันว่าไฟล์ปลอมถูกปฏิเสธ ไฟล์จริงยังผ่านปกติ

## 9. ปรับปรุงหน้าตาและการใช้งาน (UX)

### ย้าย progress bar การจองขึ้นมาไว้บนสุดแบบแนวนอน

- จากเดิมซ่อนอยู่เป็น sidebar เล็กๆ ด้านข้าง ย้ายขึ้นมาไว้บนสุดของหน้า ให้เห็นภาพรวมทั้งขั้นตอนตั้งแต่เข้าหน้าทันที

### รวมกล่อง "ข้อตกลง" กับปุ่ม "ยืนยันการจอง" เป็นก้อนเดียว

- ให้ดูเป็นขั้นตอนสุดท้ายก่อนกดยืนยันชัดเจนขึ้น แทนที่จะแยกเป็นกล่องข้อมูลลอยๆ เหมือนส่วนอื่น

### ย้าย legend สีของปฏิทินขึ้นก่อนตารางวัน

- เดิมอยู่ล่างสุดของปฏิทิน ผู้ใช้ต้องเลื่อนผ่านทุกอย่างก่อนถึงจะรู้ความหมายสี ย้ายขึ้นมาให้รู้ก่อนเริ่มเลือกวัน

### แก้ตัวเลขสรุปหน้า "การจองของฉัน" ไม่ตรงกัน

- ปัญหา: กล่องสรุปด้านบนไม่มีหมวด "เสร็จสิ้น" ทำให้บวกเลขแล้วไม่เท่ากับยอดรวมที่แสดงไว้
- แก้แล้ว: เพิ่มกล่องที่ขาดไป ตอนนี้ยอดรวมตรงกันแล้ว

### ตัดราคารวมโดยประมาณที่คลาดเคลื่อนออกจากการ์ดห้อง

- ปัญหา: หน้าค้นหาห้องเคยคำนวณ "ราคารวมต่อวัน" แบบเดายอดผิดๆ (เอาราคา/ชม. × 2 ชม. มาเรียกว่าราคาเต็มวัน) ซึ่งไม่ตรงกับราคาจริงที่คำนวณตอนจอง
- แก้แล้ว: ตัดตัวเลขที่เดาผิดได้ง่ายออก เหลือแค่ราคา/ชม. ที่แม่นยำแน่นอน ราคารวมจริงไปโชว์ตอนเลือกวัน/เวลาแทนซึ่งคำนวณถูกต้องอยู่แล้ว

## 10. ตัดการพึ่งพา CDN ภายนอก

### ระบบค้นหาที่อยู่ (ตำบล/อำเภอ/จังหวัด) พังถ้าอินเทอร์เน็ตหลุดหรือเข้า CDN ไม่ได้

- ปัญหา: library ที่ใช้ (`use-thai-address`) hardcode URL ไปดึงข้อมูลจาก jsdelivr CDN ของคนอื่นทุกครั้งที่เปิดฟอร์ม ไม่มีทาง config ให้ใช้แหล่งข้อมูลอื่นได้เลย
- แก้แล้ว: ดาวน์โหลดฐานข้อมูลที่อยู่ทั้งประเทศ (~7,500 รายการ) มาเก็บไว้ในโปรเจกต์เอง เขียน logic ถอดรหัสข้อมูลขึ้นใหม่เอง (อ้างอิงจาก logic เดิม MIT license) แล้วถอด dependency ตัวนี้ออกจากระบบทั้งหมด — ไม่มีการเรียกเน็ตออกไปหาใครเลย ข้อมูลอยู่ในเครื่อง 100%

## การไปทำต่อที่เครื่องอื่น

ทุก commit อยู่บน `origin/space-availability-check` แล้ว เครื่องอื่นดึงไปทำต่อได้ตามขั้นตอนนี้:

```bash
git checkout space-availability-check
git pull
```

### 1. ติดตั้ง dependency ใหม่

(ลบ `use-thai-address` ออกแล้ว lockfile เปลี่ยน)

```bash
npm install          # รันที่ root ของโปรเจกต์
```

ฝั่ง Go ไม่มี dependency ใหม่ ไม่ต้องทำอะไรเพิ่ม

### 2. เช็ค `.env`

⚠️ จุดสำคัญที่สุด เพราะ `.env` ไม่ได้ถูก commit เข้า git (อยู่ใน `.gitignore`) การแก้ค่าที่ทำในรอบนี้ (เพิ่ม `PROMPTPAY_ID`, `BILLER_*`, `PAYMENT_MERCHANT_*`) **จะไม่ตามไปที่เครื่องอื่นเอง**

- เครื่องใหม่ที่ยังไม่มี `.env`: `cp .env.example .env` ได้ครบทุกค่าอัตโนมัติ
- เครื่องที่มี `.env` เก่าอยู่แล้ว: เปิดเทียบกับ `.env.example` แล้วเติมส่วน `PROMPTPAY_ID` / `BILLER_ID` / `PAYMENT_MERCHANT_NAME` / `PAYMENT_MERCHANT_CITY` ที่ขาดไปเอง ไม่งั้นฟีเจอร์สร้าง QR ชำระเงินจะพัง

### 3. รัน docker + migrate ข้อมูลใหม่

```bash
docker-compose up -d
cd apps/backend && go run ./cmd/init-db
```

ขั้นตอนนี้สำคัญ เพราะโค้ด seed ที่แก้ไป (สิทธิ์ staff, ผูกอาคารให้ location) จะไม่มีผลย้อนหลังกับฐานข้อมูลเก่าจนกว่าจะรันคำสั่งนี้ใหม่ — รันซ้ำได้ปลอดภัย ไม่ลบข้อมูลเดิม (ใช้ `FirstOrCreate`)

### 4. รันตามปกติ

```bash
go run ./cmd/serve          # backend
npm run dev                 # frontend (คนละ terminal)
```

**สรุปสั้นๆ:** `git pull` → `npm install` → เช็ค `.env` → `docker-compose up -d` → `go run ./cmd/init-db` → รันทั้งสองฝั่ง
