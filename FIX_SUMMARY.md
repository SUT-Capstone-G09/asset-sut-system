# สรุปงานแก้ไขระบบ — Asset SUT System

**Branch:** `space-availability-check`
**ช่วงเวลา:** เมื่อวาน–วันนี้ (2 วัน)
**รวม:** 21 จุดที่แก้ไข ใน 9 หมวดหมู่ (backend Go + frontend Next.js)

ทุกจุดผ่านการทดสอบ build (`go build` / `go vet` / `go test`, `tsc --noEmit`) และยิงทดสอบ API/หน้าเว็บจริงก่อนสรุปว่าแก้เสร็จ ไม่ใช่แค่แก้ตามที่อ่านโค้ดเดา

---

## 1. ระบบ build ใช้งานไม่ได้เลย

- **สั่ง `npm run build` แล้วพังทันที** — bundler ตัวใหม่ (Turbopack) หา library บางตัวไม่เจอในโปรเจกต์นี้ ทั้งที่โหมด dev ใช้ปกติ เพราะ dev สั่งใช้ bundler คนละตัวกัน
  ก่อน: build ไม่ผ่านเลยสักครั้ง → ตอนนี้: ปักหมุดให้ build ใช้ bundler เดียวกับ dev ผ่านครบ 49 หน้า
- **มีไฟล์เศษหลงเหลือทำให้ build พัง** — ไฟล์ component เก่าที่ไม่มีใครเรียกใช้แล้ว ถูกวางผิดตำแหน่งจนกลายเป็นหน้าเว็บโดยไม่ตั้งใจ ลบทิ้งแล้ว
- **Type ผิดสะสมทั่วระบบ ~9 จุด** — เช่น ส่ง id ผิดชนิดข้อมูล, ส่ง field ที่ไม่มีอยู่จริงใน API, หน้าค้นหาห้องไม่รองรับการโหลดแบบ static ไล่แก้ทีละจุดจนไม่มี error เหลือ
- **หน้าแรกพังทุกครั้งที่โหลด** — รูปภาพตัวอย่างในหน้าแรกใช้ URL ที่มีอักษรไทยแปะดิบๆ โดยไม่เข้ารหัส ทำให้ browser สร้าง header สำหรับ preload รูปไม่ได้ พังทุก request

## 2. สิทธิ์การใช้งานของพนักงาน (staff)

- **พนักงาน login เข้าได้ แต่กดอะไรก็ไม่ได้เลย** — ตอน seed ข้อมูลเริ่มต้น ระบบให้สิทธิ์ (permission) กับบัญชี admin เท่านั้น บัญชี staff ไม่เคยได้รับสิทธิ์อะไรเลยสักอย่าง เพิ่มสิทธิ์ให้ staff ครบทุกงานยกเว้นการจัดการผู้ใช้ ซึ่งสงวนไว้ให้ admin เท่านั้น

## 3. ข้อมูลอาคารไม่เคยถูกบันทึกจริง

- **ระบุอาคารตอนสร้าง/แก้ไขห้องและโถง แล้วหายไปเงียบๆ** — ปัญหาซ้อนกันอยู่ 3 ชั้น: ฐานข้อมูลไม่เคยมีรายชื่ออาคารจริงเลย, ฟอร์มห้องส่งข้อมูลผิด field ไปให้ API, และฟอร์มโถงใช้รายชื่ออาคารปลอมที่ไม่ตรงกับอาคารจริงในระบบแม้แต่ตัวเดียว แก้ครบทั้ง 3 ชั้น ตอนนี้เลือกอาคารแล้วบันทึกได้จริง ดึงจากฐานข้อมูลจริง

## 4. ค่า "ศูนย์" ที่ถูกต้อง แต่ระบบดันปฏิเสธ

- **สร้างโถง (hall) ไม่ได้เลยสักครั้งเดียว** — โถงไม่มีแนวคิดเรื่อง "ความจุ" จึงส่งค่า 0 ไปเสมอ แต่ระบบตั้งกฎไว้ผิดว่าค่า 0 = "ไม่ได้กรอก" เลยปัดตกทุกครั้ง ทั้งที่ 0 คือค่าที่ถูกต้องอยู่แล้ว
- **ตั้งราคาห้องแบบให้ใช้ฟรี (0 บาท) ไม่ได้** — ติดกฎเดียวกับข้อบน แก้จุดเดียวกันทั้งสองที่

## 5. การตั้งค่าระบบ (config)

- **Log ของระบบยาวเป็นหน้ากระดาษทุกครั้งที่รัน** — ตัวแปรตั้งค่าระดับความละเอียดของ log สะกดไม่ตรงกับที่โค้ดอ่าน ทำให้การตั้งค่า "แสดงเฉพาะ error" ไม่เคยมีผลจริง ระบบพิมพ์ log ทุกคำสั่งฐานข้อมูลตลอดเวลา
- **สร้าง QR รับชำระเงินไม่ได้เลย** — ไฟล์ตั้งค่าเครื่องทดสอบขาดเลขบัญชีพร้อมเพย์ที่จำเป็นสำหรับสร้าง QR เติมค่าที่ขาดแล้วสร้าง QR ได้ปกติ

## 6. ฟอร์มสร้างเอกสารขออนุญาตใช้สถานที่

- **ช่องกรอก "วัตถุประสงค์" พิมพ์อะไรก็หายทันที** — ลืมเชื่อมช่องกรอกกับตัวจัดการค่าที่พิมพ์ ทำให้ทุกตัวอักษรที่พิมพ์ถูกล้างทิ้งทันที เชื่อมสายให้ถูกต้องแล้ว พิมพ์ได้ปกติ
- **ช่องเลือกอาคารมีตัวเลือกที่ไม่ตรงกับความจริงเลยสักอาคาร** — ตัวเลือกอาคารที่ตั้งไว้ล่วงหน้าไม่ตรงกับอาคารจริงในระบบเลยสักตัว ผู้ใช้ต้องกด "อื่นๆ" แล้วพิมพ์เองทุกครั้งอยู่ดี ตัดตัวเลือกที่ไม่มีประโยชน์ออก เหลือช่องกรอกตรงพร้อมชื่อห้องที่จองไว้ให้อัตโนมัติ
- **เพิ่มช่องกรอกเลขประจำตัวผู้เสียภาษี** (ของใหม่) — ใส่ในฟอร์มและในเอกสาร PDF ที่สร้างออกมาด้วย สำหรับหน่วยงาน/บริษัทที่ต้องใช้ออกใบกำกับภาษี

## 7. เอกสารแนบตอนยืนยันการจอง

- **เลือก "อัปโหลดเอกสาร" หรือ "สร้างเอกสารผ่านระบบ" อันเดียว แต่ติ๊กถูกทั้งคู่** — ระบบเก็บสถานะ "ใช้วิธีไหน" แยกจากลิสต์ไฟล์จริง ทำให้พอสร้างเอกสารสำเร็จ ทั้งสองการ์ดจะติ๊กถูกพร้อมกันเสมอ ปรับให้แต่ละไฟล์จำที่มาของตัวเองว่าอัปโหลดเองหรือสร้างจากระบบ แก้ปัญหาตรงจุด พร้อมเพิ่ม label บอกที่มาให้เห็นในลิสต์ไฟล์ด้วย

## 8. ลายเซ็นดิจิทัล

- **บางครั้งลายเซ็นที่เคยบันทึกไว้ "หายไป" ดื้อๆ** — ระบบมองทุกความผิดพลาด (เน็ตสะดุด, เซิร์ฟเวอร์รีสตาร์ท ฯลฯ) เป็นแบบเดียวกับ "ยังไม่เคยบันทึกลายเซ็น" หมด โดยไม่แจ้งอะไรเลย แยกสองกรณีออกจากกัน ถ้าโหลดพลาดจริงจะมีข้อความแจ้งเตือนพร้อมปุ่ม "ลองใหม่" ทั้งในหน้าโปรไฟล์และฟอร์มสร้างเอกสาร
- **[ความปลอดภัย] อัปโหลดไฟล์ปลอมอ้างว่าเป็น PNG หลอกระบบได้** — ฝั่งเซิร์ฟเวอร์เชื่อแค่ป้ายกำกับที่ผู้ใช้ส่งมาเอง ไม่เคยเปิดไฟล์ตรวจสอบจริง ทดสอบแล้วอัปโหลดไฟล์ข้อความธรรมดาแปะป้าย PNG ผ่านได้จริง เพิ่มการตรวจไฟล์จริงระดับไบต์ (ลายเซ็นไฟล์ PNG) และตรวจพื้นหลังโปร่งใสจริงฝั่งเซิร์ฟเวอร์ ปิดช่องโหว่แล้ว ทดสอบซ้ำยืนยันว่าไฟล์ปลอมถูกปฏิเสธ ไฟล์จริงยังผ่านปกติ

## 9. ปรับปรุงหน้าตาและการใช้งาน (UX)

- **ย้าย progress bar การจองขึ้นมาไว้บนสุดแบบแนวนอน** — จากเดิมซ่อนอยู่เป็น sidebar เล็กๆ ด้านข้าง ย้ายขึ้นมาไว้บนสุดของหน้า ให้เห็นภาพรวมทั้งขั้นตอนตั้งแต่เข้าหน้าทันที ตามรูปแบบมาตรฐานของหน้าจองที่คุ้นเคย
- **รวมกล่อง "ข้อตกลง" กับปุ่ม "ยืนยันการจอง" เป็นก้อนเดียว** — ให้ดูเป็นขั้นตอนสุดท้ายก่อนกดยืนยันชัดเจนขึ้น แทนที่จะแยกเป็นกล่องข้อมูลลอยๆ เหมือนส่วนอื่น
- **ย้าย legend สีของปฏิทินขึ้นก่อนตารางวัน** — เดิมอยู่ล่างสุดของปฏิทิน ผู้ใช้ต้องเลื่อนผ่านทุกอย่างก่อนถึงจะรู้ความหมายสี ย้ายขึ้นมาให้รู้ก่อนเริ่มเลือกวัน
- **แก้ตัวเลขสรุปหน้า "การจองของฉัน" ไม่ตรงกัน** — กล่องสรุปด้านบนไม่มีหมวด "เสร็จสิ้น" ทำให้บวกเลขแล้วไม่เท่ากับยอดรวมที่แสดงไว้ เพิ่มกล่องที่ขาดไป ตอนนี้ยอดรวมตรงกันแล้ว
- **ตัดราคารวมโดยประมาณที่คลาดเคลื่อนออกจากการ์ดห้อง** — หน้าค้นหาห้องเคยคำนวณ "ราคารวมต่อวัน" แบบเดายอดผิดๆ (เอาราคา/ชม. × 2 ชม. มาเรียกว่าราคาเต็มวัน) ซึ่งไม่ตรงกับราคาจริงที่คำนวณตอนจอง ตัดตัวเลขที่เดาผิดได้ง่ายออก เหลือแค่ราคา/ชม. ที่แม่นยำแน่นอน ราคารวมจริงไปโชว์ตอนเลือกวัน/เวลาแทนซึ่งคำนวณถูกต้องอยู่แล้ว
